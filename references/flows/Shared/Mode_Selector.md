# Mode_Selector 交易模式选择器

根据市场环境自动选择最适合的交易模式。

## 决策流程

```
                    ┌─────────────────┐
                    │   开盘前自检    │
                    │  (9:15-9:25)   │
                    └────────┬────────┘
                             │
                             ▼
                ┌────────────────────────┐
                │   分析市场三维度       │
                │  行情 / 情绪 / 消息    │
                └────────┬───────────────┘
                         │
          ┌──────────────┼──────────────┐
          ▼              ▼              ▼
    ┌──────────┐   ┌──────────┐   ┌──────────┐
    │ 情绪分析 │   │ 量化分析 │   │ 机构分析 │
    │涨停家数  │   │ 市场趋势 │   │ 资金流向 │
    └────┬─────┘   └────┬─────┘   └────┬─────┘
         │              │              │
         ▼              ▼              ▼
    情绪模式        量化模式        机构模式
    (Fast)          (Pro)           (Ind)
```

## 模式判断标准

### 情绪模式 (Fast) 触发条件

```
情绪模式启用：
├── 涨停家数 > 50
├── 情绪高涨（连板率>30%）
├── 热点板块明确
└── 短线资金活跃

置信度 = f(涨停家数, 连板率, 炸板率)
```

### 量化模式 (Pro) 触发条件

```
量化模式启用：
├── 市场震荡（无明显趋势）
├── 指数在均线附近波动
├── 板块轮动快
└── 追热点效果不佳

置信度 = f(波动率, 趋势性, 板块持续性)
```

### 机构模式 (Ind) 触发条件

```
机构模式启用：
├── 北向资金连续净流入 > 3天
├── 主力资金持续净买入
├── 指数上升趋势
└── 机构主导市场

置信度 = f(北向, 主力, 趋势)
```

## 选择矩阵

| 市场环境 | 情绪信号 | 资金信号 | 推荐模式 | 建议仓位 |
|----------|----------|----------|----------|----------|
| 上升 | 高涨 | 流入 | 机构(Ind) | 50-60% |
| 上升 | 高涨 | 流出 | 情绪(Fast) | 30-40% |
| 震荡 | 中性 | 中性 | 量化(Pro) | 40% |
| 震荡 | 高涨 | 流入 | 机构(Ind) | 50% |
| 下跌 | 低迷 | 流出 | 空仓 | 0% |
| 下跌 | 高涨 | 流入 | 量化(Pro) | 20% |

## 使用方法

```python
# 自动选择模式
from mode_selector import run_market_analysis

result = run_market_analysis()
# 输出推荐模式和置信度

# 指定模式
select_mode('情绪')  # 强制情绪模式
select_mode('量化')  # 强制量化模式
select_mode('机构')  # 强制机构模式
```

## 模式切换规则

```
情绪 → 量化：当情绪退潮（涨停<30家）且无趋势
量化 → 机构：当资金明显流入且指数走强
机构 → 量化：当北向流出且趋势破坏
量化 → 空仓：当市场下跌且无明确机会
```

## 注意事项

1. 模式选择每日开盘前执行一次
2. 盘中根据市场变化可临时调整
3. 模式切换需要重新进行风险评估
4. 三种模式可以同时存在（分仓操作）
